# 행사 관리 기능

## 개요

선교 행사(부스, 맛집투어, 동아리 등) 참석자를 관리하고, 유망한 참석자를 생명으로 전환하여 관리 시스템에 연결하는 기능.

- 캘린더 행사와는 별개 기능
- 대학생, 관리자 페이지에만 "행사" 탭 추가

---

## 진입 및 목록

### 탭 추가
- 대학생: [내 생명] [캘린더] **[행사]**
- 관리자: [조직도] [내 생명] [캘린더] [현황] [AI 분석] **[행사]**

### 행사 목록 (생명 목록과 동일 로직)
- 내가 연결된 행사만 표시
- 행사 카드: 행사명, 형태(일회성/동아리), 날짜, 참석자 수
- 클릭 → 행사 상세 페이지 (`/student/event/[id]`, `/manager/event/[id]`)

### 행사 추가
- [+ 행사 추가] 버튼 → 두 가지 선택:
  - **새 행사 만들기**: 행사명, 형태, 공유 대상 입력
  - **기존 행사 참가**: 행사명 검색 → 정확한 이름 입력 시 연결 (이름이 암호 역할)

### 새 행사 생성 시 입력
| 항목 | 설명 |
|------|------|
| 행사명 | 행사 이름 |
| 행사 형태 | 일회성 / 동아리 |
| 공유 대상 | 대학생·관리자 선택하여 연결 |
| 출석 집계 단위 (동아리만) | 1주일씩 집계 / 회차당 집계 |

## 행사 상세 페이지

### 탭 구성
| 탭 | 설명 |
|----|------|
| 출석 | 참석자 출석 체크 + 모니터링 |
| 상세 | 참가자 상세 정보 + 팀/관리자 배정 + 생명 전환 |
| 현황 | 통계 + 차트 + 피드백 + AI 분석 |

## 출석 탭

### 참석자 정보 (간략)
- 이름, 섭리회원 표기, 팀(배정 시), 학번(나이)

### 출석 체크
- 참석 / 미참석 체크

### 정렬
- 이름순 / 학번순 / 학과순

### 그룹 보기 (중복 적용 가능)
| 그룹 | 설명 |
|------|------|
| 기본 | 플랫 리스트 |
| 팀별 | 팀으로 묶어서 표시 |
| 관리자별 | 담당 관리자(섭리회원)로 묶어서 표시 |
| 생명만 보기 | 섭리회원 제외하고 표시 |
| 출석별 | 출석 / 미출석 구분하여 표시 |

### 히든 출석 페이지 (동아리용)

행사 생성 시 자동으로 출석 체크 URL 생성 (예: `cnucare.vercel.app/check/러닝`)
- 로그인 없이 접근 가능한 별도 페이지
- 각자가 현장에서 직접 출석
- 입력: 이름, 날짜(기본 오늘, 변경 가능), 함께 띈 사람 이름 추가(+버튼으로 여러 명)
- 출석 완료 시 본인 + 함께 띈 사람 전부 같은 날 같은 그룹으로 출석 기록
- 같은 날 같은 그룹 = 같은 팀으로 자동 인식 (별도 팀 배정 불필요)
- 체크 완료 시:
  - 행사 출석 탭에 자동 반영
  - 생명으로 등록된 참가자는 일지 자동 생성 (목적=관리, 날짜=출석일, 장소=행사명, 반응="[행사명] 출석완료")

### 동아리 출석 (회차별)

#### 주차별 보기
요일마다 배정된 사람이 다름. 팀별로 묶어서 표시.

```
2주차
  월(4/7)           화(4/8)          수(4/9)
  [1팀]             [1팀]            [1팀]
  김서현 ✓          박지민 ✓         도연희 ✓
  이은서 ✓          강현   -         김나영 ✓
  [2팀]             [2팀]
  조유현 -          곽도현 ✓
  나환주 ✓          명주완 ✓
```

#### 전체 보기
주차별 보기가 아래로 쌓여서 1주차, 2주차, 3주차... 순서로 표시.

#### 출석률 보기
버튼 클릭 시 사람별 통계:
- 이름, 출석 횟수/총 횟수, 출석률(%), 게이지바

## 상세 탭

### 참가자 상세 정보 (PDF 형태)
- 이름, 성별(남/여), 학과, 학번, 함께 신청한 친구, 메모
- 클릭 시: 연락처 등 추가 정보 펼침

### 행사 전/후 모드 전환
| 모드 | 주요 기능 |
|------|----------|
| 행사 전 | 팀 매칭, 관리자(섭리회원) 배정 |
| 행사 후 | 관리자 피드백 작성, 생명 전환(Pass/Fail) |

### 배정 (편집 가능)
- 관리자(섭리회원) 배정
- 팀 배정
- 배정 없이도 행사 운영 가능 (배정 안 한 참가자 = 덜 중요한 참가자)

### 생명 전환 (생명 목록에서 처리)
- 행사 상세에서 직접 전환하지 않음
- **생명 목록에서 이름 넣고 추가** → 행사 참석자에 매칭되는 사람이 있으면 "[행사명] 김서현이 맞나요?" + 학과/학번 표시, 동명이인이면 후보 여러 명 보여주고 선택
- 확인 후 추가 → 생명 시트 생성
  - 기본 정보(이름, 성별, 학과, 학번 등)는 행사 시트에서 자동으로 가져옴
  - 없는 정보는 비워두고 나중에 채우기
  - 특이사항에 "[행사명] 참여" 자동 입력
- 행사 상세에서는 Pass/Fail 상태만 관리 (생명 전환 여부 표시용)

### 정렬 + 그룹
- 출석 탭과 동일 + **친구별** 그룹 추가
- 친구별: 함께 신청한 사람 묶기 + 학과별 상위 그룹 (PDF 참고)

## 현황 탭

### 통계
- 총 신청 인원
- 총 출석 인원
- 남/여 비율
- 생명 전환 수

### 차트
- 학년별 출석 인원 (막대 차트)
- 학과별 인원 (막대 차트)

### 피드백 (히든 피드백 페이지)
- "피드백 생성" 버튼 → 설정 후 URL 생성
- 설정: 익명/실명 선택, 질문 항목 편집 (기본: 좋았던 점, 아쉬웠던 점 + 커스텀 추가)
- 생성된 URL (`/feedback/[id]`): 로그인 없이 접근, 실명이면 이름 입력, 각 질문에 텍스트 답변
- 제출된 피드백은 현황 탭에서 확인 가능

### AI 분석
- 분석 전 입력: 행사 기간(예: 3일), 총 비용
- 분석 데이터: 참석자 데이터 + 섭리회원 정보 + 생명 전환율 + 피드백 전부 포함
- 평가 기준:
  - 대규모 + 높은 전환율 = 좋은 행사
  - 대규모 + 낮은 전환율 = 힘만 뺀 행사
  - 소규모 + 모집 부진 = 아이템 점검 필요
  - 목적성 소규모 + 낮은 전환율 = 아쉬운 행사
  - 비용 대비 효율 평가
  - 개선점 + 향후 전략 제안

## 동아리 — 상세/현황 탭

일회성과 동일한 구조이며, 아래 차이점만 다름.

### 상세 탭
- 일회성과 동일 (참가자 상세, 팀/관리자 배정, Pass/Fail, 친구별 그룹)
- 추가: 요일별 배정 관리 (월~금 중 어떤 요일에 나오는지)
- 행사 전/후 모드 → 동아리는 항상 "진행 중" 상태로, 팀 매칭/피드백/Pass/Fail 상시 가능

### 현황 탭
- 통계: 총 등록 인원, 현재 활동 인원, 이탈 인원, 남/여 비율, 생명 전환 수
- 차트: 학년별 인원, 학과별 인원, **회차별 출석 추이** (꺾은선 차트)
- 피드백: 일회성과 동일 (생명용 / 섭리회원용)
- AI 분석: 일회성과 동일 + 출석 패턴 분석 (개근자, 이탈 위험자, 출석률 하락 추세 등)

## 용어 정리

| 용어 | 설명 |
|------|------|
| 섭리회원 | 교회 사람들 (대학생, 관리자, 지도자 등) |
| 생명 전환 | 행사 참석자를 생명 목록으로 넘기는 것. 다음 약속을 잡았거나 전초가 가능한 수준으로 친해진 인원 |
| Pass | 생명 전환 대상 |
| Fail | 전환 안 함 |
| 친구별 | 함께 신청한 사람 그룹 |

## 예상 DB 테이블

### events
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| name | text | 행사명 |
| type | text | "onetime" / "club" |
| club_unit | text | 동아리일 때: "daily" / "weekly" |
| created_by | uuid | 생성자 |
| created_at | timestamp | |

### event_members (행사 참여 섭리회원)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| event_id | uuid | FK→events |
| user_id | uuid | FK→users |

### event_attendees (행사 참석자)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| event_id | uuid | FK→events |
| name | text | 이름 |
| gender | text | "남" / "여" |
| department | text | 학과 |
| year | int | 학번(입학년도) |
| phone | text | 연락처 |
| team | text | 팀 배정 |
| manager_id | uuid | 담당 섭리회원 |
| friend_group | text | 함께 신청한 친구 그룹명 |
| status | text | "pending" / "pass" / "fail" |
| memo | text | 메모 |
| life_id | uuid | 생명 전환 시 연결된 life ID |
| created_at | timestamp | |

### event_attendance (출석 기록)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| event_id | uuid | FK→events |
| attendee_id | uuid | FK→event_attendees |
| date | date | 출석 날짜 |
| present | boolean | 출석 여부 |

### event_feedback (피드백)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| event_id | uuid | FK→events |
| author_id | uuid | 작성자 (null이면 생명용) |
| content | text | 피드백 내용 |
| type | text | "life" / "member" |
