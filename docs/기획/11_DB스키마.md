# DB 스키마 설계

## 테이블 목록

1. users (사용자)
2. lives (생명)
3. user_lives (사용자-생명 연결)
4. journals (일지)
5. lessons (강의 마스터)
6. lesson_checks (강의 체크)
7. worship_attendance (예배 참석)
8. bible_reading (성경 읽기)
9. audio_queue (녹음 변환 작업 큐)

---

## 1. users (사용자)

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | uuid | PK | 고유 ID |
| login_id | text | 필수, unique | 로그인 아이디 |
| password | text | 필수 | 4자리 비밀번호 (해시 저장) |
| name | text | 필수 | 실명 |
| birth_date | date | 필수 | 생년월일 |
| phone | text | 필수 | 전화번호 |
| role | text | 필수 | 역할: student / manager / instructor / admin |
| display_name | text | 자동생성 | 표시 이름 (예: 박수훈00) |
| manager_id | uuid | 선택, FK→users | 소속 관리자 (대학생만 해당) |
| created_at | timestamp | 자동 | 가입일 |

---

## 2. lives (생명)

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | uuid | PK | 고유 ID |
| name | text | 필수 | 생명 이름 |
| student_id_number | text | 선택 | 학번 |
| age | int | 선택 | 나이 |
| grade | text | 선택 | 학년 |
| department | text | 선택 | 학과 |
| mbti | text | 선택 | MBTI |
| meeting_reason | text | 선택 | 만남 경위 |
| meeting_count | int | 기본값 0 | 만남 횟수 |
| has_partner | boolean | 선택 | 이성 유무 |
| characteristics | text | 선택 | 특징 (본가/가족관계/종교/통학 등 자유 서술) |
| birth_year | text | 선택 | 생년 뒤 2자리 (표시용: 이름+생년) |
| stage | text | 필수 | 현재 단계 (아래 참고) |
| is_failed | boolean | 기본값 false | 페일 여부 |
| last_met_at | date | 선택 | 마지막 만남 날짜 |
| created_at | timestamp | 자동 | 등록일 |
| updated_at | timestamp | 자동 | 최근 업데이트일 |

### stage 값 정의
| 값 | 설명 |
|----|------|
| first_meeting | 1차 만남 |
| pre_visit | 전초 (관리자 연결) |
| intro | 입문 (강사 연결, 1~5강) |
| beginner | 초급 (6~10강) |
| intermediate | 중급 (11~19강) |
| advanced | 고급 (20~30강) |
| completed | 수료 |

## 3. user_lives (사용자-생명 연결, N:N)

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | uuid | PK | 고유 ID |
| user_id | uuid | FK→users | 사용자 |
| life_id | uuid | FK→lives | 생명 |
| role_in_life | text | 필수 | 이 생명에서의 역할: evangelist / manager / instructor |
| created_at | timestamp | 자동 | 연결일 |

> unique(user_id, life_id) — 같은 사용자가 같은 생명에 중복 연결 불가

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | uuid | PK | 고유 ID |
| life_id | uuid | FK→lives | 생명 |

## 5. lessons (강의 마스터)

시스템에 미리 등록되는 30개 강의 + 부가 과정 목록.

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | uuid | PK | 고유 ID |
| number | int | 필수 | 강의 번호 (1~30) |
| name | text | 필수 | 강의명 |
| level | text | 필수 | 등급: intro / beginner / intermediate / advanced |
| category | text | 필수 | 구분: lecture / prayer_edu / worship_edu / criticism_edu / history |
| sort_order | int | 필수 | 정렬 순서 |

## 9. audio_queue (녹음 변환 작업 큐)

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | uuid | PK | 고유 ID |

| completed_at | timestamp | 선택 | 완료 시각 |

## 테이블 관계도

```
users (사용자)
  │
  ├── manager_id → users (대학생의 소속 관리자)
  │
  ├── user_lives (N:N) → lives (생명)
  │
  ├── journals.author_id (일지 작성자)
  │
  └── audio_queue.requester_id (녹음 변환 요청자)

lives (생명)
  │
  ├── user_lives (N:N) → users (연결된 사용자들)
  │
  ├── journals (일지 목록)
  │
  ├── lesson_checks → lessons (강의 체크)
  │
  ├── worship_attendance (예배 참석)
  │
  ├── bible_reading (성경 읽기)
  │
  └── audio_queue (녹음 변환 대기열)

lessons (강의 마스터, 고정 데이터)
  │
  └── lesson_checks (강의 체크 기록)
```

## 테스트 데이터 계획

| 역할 | 인원 | 비밀번호 |
|------|------|----------|
| 강사 | 1명 | 1234 |
| 관리자 | 4명 | 1234 |
| 대학생 | 20명 | 1234 |
| 어드민 | 1명 | 1234 |

- 강의 마스터 데이터 30개 + 부가 과정 미리 등록
- 관리자별로 대학생 5명씩 배정
- 이름은 임의 한국 이름으로 생성
