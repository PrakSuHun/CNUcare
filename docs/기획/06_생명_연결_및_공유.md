# 생명 연결 및 공유 체계

## 생명 가시성 원칙

> 대학생은 **자신에게 연결된 생명만** 자기 페이지에서 볼 수 있다.
> 같은 생명에 여러 대학생이 연결되어 있으면, 모두가 해당 생명 페이지를 열람·관리·기록할 수 있다.

---

## 생명 등록 시 중복 확인 프로세스

대학생이 생명을 새로 등록할 때, 이미 다른 대학생이 등록한 동일 인물일 수 있으므로 중복 확인을 거친다.

### 흐름

```
[생명 추가] 클릭
    ↓
이름 입력
    ↓
DB에서 이름 유사 검색
    ↓
┌─ 유사한 생명이 있음 ──────────────────────┐
│                                            │
│  "이 사람인가요?" (기본 정보 함께 표시)     │
│                                            │
│  [예, 이 사람입니다]    [아니요, 새로 등록] │
└────────────────────────────────────────────┘
    ↓                         ↓
기존 생명에 연결            새 생명 생성 페이지로 이동
(내 목록에 추가됨)          (기본 정보 입력)
```

### "예, 이 사람입니다" 선택 시
- 기존 생명에 현재 대학생이 **추가 연결**됨
- 내 생명 목록에 해당 생명이 나타남
- 기존에 작성된 일지, 기본 정보 모두 열람 가능
- 기존 연결된 다른 대학생·관리자·강사와 **함께 관리**

### "아니요, 새로 등록" 선택 시
- 일반 생명 생성 프로세스 진행 (이름만으로 생성 가능)

### 유사 검색 기준
- 이름이 동일하거나 유사한 생명을 DB에서 조회
- 후보가 여러 명이면 목록으로 표시 (기본 정보로 구분 가능하게)
- 후보가 없으면 바로 새 생명 생성 페이지로 이동

---

## 생명-사용자 연결 구조

하나의 생명에 여러 사용자가 연결될 수 있는 **다대다(N:N) 관계**:

```
생명 A ─── 대학생 1 (최초 등록자)
       ├── 대학생 2 (같이 전도)
       ├── 관리자 1 (전초 담당)
       └── 강사 1 (말씀 담당)
```

- 연결된 모든 사용자가 동일한 수정/작성 권한을 가짐
- 누가 연결되어 있는지 생명 상세 페이지에서 확인 가능

## 미결 사항
- [ ] 관리자/강사는 어떻게 생명에 연결되는지 (대학생이 지정? 관리자가 직접 검색해서 연결? 별도 "패스" UI?)
- [ ] 연결 해제가 가능한지 (더 이상 담당하지 않을 때)
- [ ] 유사 검색 시 이름 외에 학과/학번 등도 비교 조건으로 사용할지
