# AI 분석 보고서

## 개요
Gemini API를 활용하여 누적된 데이터를 분석하고, 4가지 레벨의 보고서를 생성한다.
보고서는 저장되어 나중에 열람 가능.

---

## 보고서 유형

### 1. 생명별 분석
- 대상: 개별 생명 1명
- 입력 데이터: 기본 정보 + 전체 일지 + 강의 진도
- 분석 내용:
  - 생명의 현재 영적 상태 요약
  - 최근 반응 동향 (긍정적/부정적 변화)
  - 성격·소통 스타일 파악
  - 관심사와 고민 파악
  - 주의사항 (민감한 주제, 하면 안 되는 말)
  - 다음 단계 제안 (어떻게 접근하면 좋을지)

### 2. 대학생별 분석
- 대상: 대학생 1명 + 연결된 모든 생명
- 입력 데이터: 대학생의 모든 생명 정보 + 일지 + 진도 + 페일 이력
- 분석 내용:
  - 전도 스타일 장점/단점
  - 어떤 유형의 생명과 잘 맞는지
  - 페일 패턴 분석 (어떤 상황에서 페일이 발생하는지)
  - 성장 포인트 제안

### 3. 관리자 팀 분석
- 대상: 관리자 1명 + 소속 대학생들 + 생명들
- 입력 데이터: 관리자 소속 전체 데이터
- 분석 내용:
  - 팀 전체 현황 요약
  - 대학생별 성과 비교
  - 팀워크 분석
  - 병목 구간 파악
  - 개선 제안

### 4. 전체 분석
- 대상: 전체 조직 데이터
- 입력 데이터: 모든 생명, 일지, 진도, 사용자 데이터
- 분석 내용:
  - 전체 전도 현황 요약
  - 단계별 병목 분석
  - 성공/실패 패턴
  - 관리자별 성과 비교
  - 전략적 개선 제안

---

## UI 흐름

```
분석 탭 → 보고서 유형 선택 → 대상 선택 → [분석 시작] → 생성 중... → 보고서 표시
                                                                    ↓
                                                              저장 (DB)
                                                                    ↓
                                                         기존 보고서 목록에서 열람
```

---

## DB: reports 테이블
| 필드 | 설명 |
|------|------|
| type | life / student / manager / overall |
| target_id | 분석 대상 ID (생명/대학생/관리자 ID, 전체는 null) |
| target_name | 대상 이름 (표시용) |
| content | 보고서 본문 (마크다운) |
| created_at | 생성일 |
