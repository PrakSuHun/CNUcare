# 일지 양식 및 권한 체계

## 일지 양식

| 필드 | 필수 여부 | 설명 |
|------|----------|------|
| 날짜 | 필수 | 만남 또는 기록 날짜 |
| 만남 장소 | 필수 | 어디서 만났는지 |
| 생명 반응 | 필수 | 자유 서술 영역. 아래 내용을 포함하여 작성: |

### 생명 반응에 포함할 내용
- 어떤 대화를 했는지
- 생명의 반응은 어떠했는지
- 다음 계획은 무엇인지
- 기타 특이사항

> 별도 항목으로 분리하지 않고, **생명 반응 하나의 텍스트 필드**에 자유롭게 서술한다.
> 녹음 → Gemini 변환 시에도 이 세 필드(날짜, 장소, 반응)로 정리되도록 프롬프트를 구성한다.

---

## 단계 전환 방식

- **수동 전환**: 대학생 또는 관리자가 직접 단계를 변경할 수 있음
- 자동 승인 절차 없음 (별도 수락/거절 프로세스 없음)
- 강의 레벨(초급~수료)은 강의 체크 기록에 따라 **자동 반영**

---

## 권한 체계

### 핵심 원칙
> 해당 생명에 **연결된** 대학생·관리자·강사는 모두 동일한 데이터 수정 및 작성 권한을 가진다.

| 권한 | 대학생 | 관리자 | 강사 |
|------|--------|--------|------|
| 생명 기본 정보 수정 | O | O | O |
| 일지 작성 | O | O | O |
| 일지 수정 | O | O | O |
| 단계 변경 | O | O | - |
| 강의 체크 | - | - | O |
| 페일 처리 | O | O | O |

> 단, 위 권한은 **자신에게 연결된 생명**에 한정된다.

---

## 강의 레벨 체계 (추후 확정)

- 강의명과 등급(초급/중급/고급)은 추후 사용자가 제공 예정
- 강사가 강의 체크를 기록하면, 해당 등급의 강의 수강 완료 수에 따라 레벨이 자동 반영됨
- 구체적인 강의 목록과 레벨 전환 기준은 데이터 제공 후 확정
