# 대학생 화면 흐름 상세

## 1. 로그인 페이지
- 이름 입력으로 간편 로그인

---

## 2. 생명 목록 페이지

생명 추가 후 이 페이지로 돌아온다.

### 목록 항목에 표시되는 정보
| 항목 | 설명 |
|------|------|
| 생명 이름 | 클릭 시 생명 상세 페이지로 이동 |
| 최근 업데이트 날짜 | 마지막 일지 작성 또는 정보 수정 날짜 |
| 현재 단계 | 아래 단계 정의 참고 |

### 생명 단계 정의

```
1차 만남 → 관리자 연결 → 강사 연결 → 초급 → 중급 → 고급 → 수료
```

- **1차 만남**: 대학생이 생명을 등록한 초기 상태
- **관리자 연결**: 관리자에게 패스되어 전초 진행 중
- **강사 연결**: 강사에게 연결되어 말씀 강의 시작
- **초급 → 중급 → 고급 → 수료**: 강사 연결 이후, 강의 수강 진행도에 따라 자동 변경 (강의 체크 반영)

### 목록 구성
- **상단**: 활성 생명 목록 (진행 중인 생명들)
- **하단**: 페일 목록 (포기한 생명들, 눈에 잘 띄지 않게 배치)

### 버튼
- **[+ 생명 추가]** → 생명 생성 페이지로 이동

## 3. 생명 상세 페이지

생명 이름 클릭 시 진입.

### 페이지 구성
1. **기본 정보 영역**: 지금까지 작성한 생명 기본 정보 표시 (수정 가능)
2. **일지 목록 영역**: 작성된 일지가 시간순으로 누적 표시
   - 기존 일지 클릭 → 수정 가능
3. **액션 버튼**:
   - **[+ 일지 추가]** → 일지 작성 페이지로 이동
   - **[페일]** → 활성화 시 생명이 페일 목록으로 이동

### 페일 처리
- 페일 버튼을 누르면 해당 생명은 목록 페이지에서 하단 페일 목록으로 이동
- 더 이상 진행하지 않는 생명을 정리하는 용도
- 페일된 생명은 중요하지 않으므로 시각적으로 축소/접힘 처리

---

## 4. 일지 작성 페이지

일지 추가 버튼 클릭 시 두 가지 작성 방식 선택:

### 방식 1: 직접 작성
- 텍스트로 일지 양식에 맞춰 직접 입력

### 방식 2: 녹음 작성
- **실시간 녹음**: 앱에서 바로 녹음
- **녹음 파일 업로드**: 기존 녹음본 첨부
- 녹음 파일 → **Supabase Storage**에 저장
- 녹음 파일 → **Gemini 무료 API**로 전송 → 일지 양식에 맞춰 텍스트 변환·정리
- 변환된 텍스트를 사용자가 확인/수정 후 저장

---

## 미결 사항
- [ ] 일지 양식은 어떤 항목으로 구성되는지 (예: 날짜, 만남 장소, 대화 내용, 생명 반응, 다음 계획 등)
- [ ] 페일된 생명을 다시 활성화할 수 있는지 (복구 기능)
- [ ] 단계 변경은 자동인지 수동인지 (예: 관리자 연결은 관리자가 수락해야 변경?)
- [ ] 강의 레벨(초급~수료) 전환 기준 (몇 개 강의를 들으면 다음 레벨?)
- [ ] 녹음 텍스트 변환 후 사용자 확인 단계가 필요한지
- [ ] 생명 기본 정보도 이 상세 페이지에서 수정 가능한지
