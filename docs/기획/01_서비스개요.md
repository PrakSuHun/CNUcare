# CNUcare 서비스 개요

## 프로젝트 목적
교회 선교 활동에서 **생명**(선교 대상자)의 등록부터 전초, 말씀 교육까지의 전 과정을 관리하고, 다수의 생명 진행 상황을 한눈에 파악할 수 있는 웹 서비스.

---

## 사용자 역할 (3종)

| 역할 | 설명 |
|------|------|
| **대학생** | 선교 현장에서 생명을 만나고 등록하는 역할. 생명의 기본 정보를 최초 등록하고, 이후 일지를 통해 지속적으로 상태를 공유한다. |
| **관리자** | 대학생이 등록한 생명을 패스받아 **전초**(사전 만남/탐방)를 진행하는 역할. 전초 결과와 생명의 반응을 일지로 기록·공유한다. |
| **강사** | 전초가 완료된 생명에게 **말씀**(성경 강의)을 가르치는 역할. 생명의 일지를 파악하여 강의를 진행하고, 어떤 강의를 들었는지 체크한다. |

---

## 핵심 엔티티

### 생명 (Life)
- 선교 대상자 한 명을 의미
- 대학생이 최초 등록 (기본 정보 입력)
- 이후 대학생 → 관리자 → 강사 순으로 관여하며 단계가 진행됨
- **한 사용자가 여러 생명을 병렬로 담당**할 수 있음

### 일지 (Journal)
- 생명에 대한 기록을 시간순으로 쌓아가는 로그
- 대학생, 관리자 모두 작성 가능
- 작성 상황 예시:
  - 대학생: 생명을 처음 만난 이야기, 최신 근황, 전초 전 정보 공유
  - 관리자: 전초 방문 결과, 생명의 반응, 특이사항
  - 강사: (열람 중심) 강의 준비를 위한 생명 파악

### 강의 체크 (Lesson Check)
- 강사가 생명에게 강의를 진행한 후, 어떤 강의를 들었는지 체크하는 기록
- 강의 커리큘럼 기반으로 진행 상태 추적

---

## 핵심 워크플로우

```
대학생: 생명 등록 (기본 정보)
    ↓
대학생: 일지 작성 (만남 기록, 근황 공유)
    ↓
관리자: 생명 패스받음 → 전초 진행
    ↓
관리자: 일지 작성 (전초 결과, 반응 기록)
    ↓
강사: 일지 열람 → 생명 파악 → 말씀 강의 진행
    ↓
강사: 강의 체크 (어떤 강의를 들었는지 기록)
```

---

## 핵심 가치
> **"한눈에 보이는 진행 상황"**
> 
> 대학생·관리자·강사 모두 다수의 생명을 병렬로 담당하므로,
> 각 생명의 현재 단계와 최근 일지를 **대시보드 형태로 한눈에 파악**할 수 있어야 한다.

## 미결 사항 (추후 기획 필요)
- [ ] 생명의 기본 정보에 어떤 필드가 들어가는지
- [ ] 생명의 단계(상태) 정의 (예: 등록 → 전초중 → 말씀중 → 완료?)
- [ ] 역할별 권한 범위 (대학생이 다른 대학생의 생명을 볼 수 있는지 등)
- [ ] 강의 커리큘럼 구조 (몇 개의 강의가 있고, 순서가 있는지)
- [ ] 생명의 "패스" 프로세스 상세 (자동/수동, 승인 필요 여부)
- [ ] 알림 기능 필요 여부
- [ ] 모바일 대응 필요 여부
