# 생명 기본 정보

## 필드 정의

기존 스프레드시트 기반으로 정리. **이름만 입력하면 생성 가능**하며, 나머지는 파악되는 대로 점진적으로 작성한다.

| 필드 | 필수 여부 | 설명 |
|------|----------|------|
| 이름 | **필수** | 생명의 이름 (생성 시 최소 조건) |
| 학번(나이) | 선택 | 학번 또는 나이 |
| 학년 | 선택 | 현재 학년 |
| 학과 | 선택 | 소속 학과 |
| MBTI | 선택 | 성격 유형 |
| 관리자 | 선택 | 담당 관리자 이름 |
| 만남 경위 | 선택 | 어떻게 만나게 되었는지 |
| 만남 횟수 | 선택 | 만남 횟수 (자동 카운트 가능성 검토) |
| 이성 유무 | 선택 | 연인 유무 |
| 특징 | 선택 | 본가/가족관계/종교/MBTI/통학 여부 등 자유 서술 |

## 대학생 기준 화면 흐름

### 1. 로그인 페이지
- 자기 이름을 입력하여 로그인
- 별도 비밀번호 없음 (이름 기반 간편 로그인)

### 2. 생명 목록 페이지
- 자신이 등록한 생명들의 이름 목록 표시
- **[+ 생명 추가]** 버튼으로 새 생명 생성

### 3. 생명 생성 페이지
- 위 필드를 입력하는 폼
- **이름만 입력하면 생성 가능** (나머지는 나중에 채워도 됨)
- 생성 완료 후 → 해당 생명의 상세 페이지로 이동

## 미결 사항
- [ ] 로그인 시 이름이 겹치면 어떻게 구분하는지 (동명이인 처리)
- [ ] 만남 횟수는 수동 입력인지, 일지 작성 시 자동 카운트인지
- [ ] 생명 목록에서 이름 외에 어떤 정보를 함께 보여줄지 (단계, 최근 일지 날짜 등)
- [ ] 생명 상세 페이지에서 기본 정보 수정이 가능한지
